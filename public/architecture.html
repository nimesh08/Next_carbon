3. Token Lifecycle

stateDiagram-v2 [*] --> Purchase: User pays via Razorpay Purchase --> RTP_Minted: Mint ProjectToken on-chain RTP_Minted --> Pooled: Deposit to CreditPool Pooled --> SEC_Held: Receive weighted SEC tokens RTP_Minted --> PartialMature: Admin triggers X% yearly PartialMature --> ACC_Partial: Burn RTP + Mint ACC ACC_Partial --> PartialMature: Next year cycle ACC_Partial --> ACC_Full: 100% matured SEC_Held --> ACC_Claimed: Claim after maturity ACC_Full --> Offset: Retire credits ACC_Claimed --> Offset: Retire credits Offset --> Burned: Permanent removal Burned --> [*] RTP_Minted --> Redeemed: Withdraw to wallet ACC_Full --> Redeemed: Withdraw to wallet

Phase 1: Purchase (RTP Minting)

1
User pays via Razorpay — INR payment for shares of a carbon project
2
Backend verifies payment — Razorpay signature check
3
ON-CHAIN: ProjectToken.mint(companyWallet, shares) — RTP minted
4
DB: user_token_balances RTP += shares

Phase 2: Pool (Optional)

5
ON-CHAIN: CreditPool.deposit(tokenAddr, amount) — RTP to pool, SEC minted (amount * weight)

Phase 3: Partial Yearly Maturity

6
Admin triggers X% — e.g. 20% per year for 5 years
7
ON-CHAIN: ProjectToken.burnFrom(companyWallet, partialAmt)
8
ON-CHAIN: CreditManager.mintAcc(companyWallet, partialAmt)
9
DB: Each user RTP reduced, ACC increased proportionally

Phase 4: Offset / Retirement

10
ON-CHAIN: CreditManager.offset(holder, amount) — ACC burned forever
11
PDF retirement certificate generated with tx hash

Phase 5: Redeem to Wallet

12
ON-CHAIN: ProjectToken.transfer(userWallet, amount) — real ERC-20 transfer

User Actions
  • Sign Up → Supabase Auth
  • KYC → Upload doc, admin approves
  • Buy Shares → Razorpay + RTP mint
  • View Portfolio → Shares + map + tokens
  • Pool Tokens → RTP to SEC
  • Offset → Burn ACC permanently
  • Redeem → Tokens to personal wallet
Admin Actions
  • Create Project → Deploys ERC-20
  • Manage Projects → Update details
  • Approve KYC → Review documents
  • Partial Maturity → Yearly burn+mint
  • Monitor Users → All users + KYC
