3. Token Lifecycle

Phase 1: Purchase (Project Token Minting)

1
User pays via Razorpay — INR payment for shares of a carbon project
2
Backend verifies payment — Razorpay signature check
3
ON-CHAIN: ProjectToken.mint(companyWallet, shares) — PT minted
4
DB: user_token_balances PT += shares

Phase 2: Diversification Pool (Optional)

5
ON-CHAIN: DiversificationPool.deposit(tokenAddr, amount) — PT to pool, CIT minted (amount × weight)

Phase 3: Partial Yearly Maturity

6
Admin triggers X% — e.g. 20% per year for 5 years
7
ON-CHAIN: ProjectToken.burnFrom(companyWallet, partialAmt)
8
ON-CHAIN: CarbonLifecycleManager.mintVcc(companyWallet, partialAmt)
9
DB: Each user PT reduced, VCC increased proportionally

Phase 4: Carbon Offset / Retirement

10
ON-CHAIN: CarbonLifecycleManager.offset(holder, amount) — VCC burned forever
11
PDF retirement certificate generated with tx hash

Phase 5: Redeem to Personal Wallet

12
ON-CHAIN: ProjectToken.transfer(userWallet, amount) — real ERC-20 transfer

6. Token Summary

Project Token (PT)

Represents fractional ownership in a specific carbon project. Each project has its own unique ERC-20 token (e.g., CEARR for Brazil Biomass). Minted when users purchase shares. Can be deposited to pool or held until maturity.

Carbon Index Token (CIT)

Represents diversified exposure across multiple projects. Received when depositing PT to the Diversification Pool. Amount received = PT × project weight. Can be withdrawn for proportional PT or claimed for VCC after maturity.

Verified Carbon Credit (VCC)

Represents real, verified carbon credits. Created through yearly partial maturity (PT → VCC conversion). Can be retired (burned) for carbon offsetting with PDF certificate, or redeemed to personal wallet.
